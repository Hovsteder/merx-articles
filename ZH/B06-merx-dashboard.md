# MERX 仪表板：无需写代码即可交易能量

并非每个能量购买者都是开发者。并非每个开发者都想为一次性购买编写代码。也并非每个组织都有工程带宽在集成 API 之后才能开始节省 TRON 交易成本。

MERX 仪表板位于 [merx.exchange](https://merx.exchange)，提供完整的 Web 界面用于购买能量、比较供应商价格、管理余额、追踪订单和生成 API 密钥——无需编写任何代码。本文详细介绍仪表板的每个功能并展示如何高效使用它。

## 快速上手

访问 [merx.exchange](https://merx.exchange) 并创建账户。注册只需要电子邮件地址和密码。无 KYC。无身份验证。无等待期。你的账户立即激活。

登录后，你会进入主仪表板视图。界面采用深色主题和高对比度排版——没有视觉噪音，没有不必要的装饰，只有你做出购买决策所需的信息。

## 价格面板

你在仪表板上看到的第一件事是实时价格面板。它显示来自所有七个集成供应商的当前能量价格，每 30 秒更新一次。

```
Provider      1h Price    1d Price    Available
-------------------------------------------------
Feee          28 SUN      22 SUN      Yes
Netts         31 SUN      24 SUN      Yes
itrx          32 SUN      23 SUN      Yes
CatFee        30 SUN      25 SUN      Yes
PowerSun      33 SUN      26 SUN      Yes
TronSave      35 SUN      28 SUN      Yes
SoHu          34 SUN      27 SUN      Yes
```

面板为每个时长层级高亮显示最便宜的供应商。价格以每能量单位 SUN 显示——这是允许跨供应商直接比较的标准计价单位，无论它们内部如何报价。

### 价格含义

每个价格代表指定租赁时长的每单位能量成本。要计算你购买的总成本：

```
Total cost = energy_amount * price_per_unit

Example:
  65,000 energy * 28 SUN/unit = 1,820,000 SUN = 1.82 TRX
```

当你输入订单时，仪表板会自动执行此计算。你在确认前会看到以 SUN 和 TRX 表示的总成本。

### 价格历史

实时价格下方有一个历史图表，显示过去 24 小时、7 天或 30 天的价格走势。这帮助你识别规律——价格往往在非高峰时段（UTC 00:00-08:00）较低，在高峰交易时段较高。

图表不仅仅是信息展示。如果你下大额订单且时间上有灵活性，等待几小时等价格回落可以节省可观的比例。

## 创建订单

订单创建表单是仪表板的核心功能。流程如下：

### 步骤 1：输入订单参数

填写三个字段：

- **能量数量**：你需要的能量单位数。如果不确定，仪表板提供常用数量的快速选择按钮：
  - 32,000（最低 USDT 转账）
  - 65,000（标准 USDT 转账）
  - 200,000（DEX 交换）
  - 500,000（复杂合约交互）
  - 自定义数量

- **目标地址**：将接收能量委托的 TRON 地址。这是将执行需要能量的交易的地址。仪表板在继续前验证地址格式。

- **时长**：你需要能量的时间。选项通常包括 1 小时、1 天、3 天、7 天、14 天和 30 天。

### 步骤 2：查看报价

输入参数后，仪表板显示详细报价：

```
Order Summary
-------------------------------------------------
Energy:           65,000 units
Duration:         1 hour
Target:           TYourAddress...
Best provider:    Feee
Price per unit:   28 SUN
Total cost:       1,820,000 SUN (1.82 TRX)
Account balance:  50.00 TRX
Balance after:    48.18 TRX
```

报价精确显示哪个供应商将完成订单、每单位价格、总成本以及对你账户余额的影响。没有隐藏费用或加价。

### 步骤 3：确认并执行

点击确认按钮。订单提交到 MERX 后端，后端将其路由到选定的供应商执行。委托通常在几秒内完成。

订单完成后，仪表板更新显示订单状态、链上委托交易哈希和租赁剩余时间。

## 管理余额

### 充值

在购买能量之前，你需要在 MERX 上有 TRX 余额。充值流程：

1. 导航到余额部分
2. 点击"充值"
3. 仪表板显示你的专属充值地址
4. 从任何 TRON 钱包向该地址发送 TRX
5. 链上确认后自动到账

充值监控服务持续监视入账交易。通常在交易在链上确认后 1-2 分钟内到账。

### 查看余额历史

余额部分显示所有余额变动的完整历史：

```
Date/Time            Type          Amount       Balance
-----------------------------------------------------------
2026-03-28 14:30     Deposit       +100.00 TRX  100.00 TRX
2026-03-28 14:35     Order #1247   -1.82 TRX    98.18 TRX
2026-03-28 16:20     Order #1248   -1.95 TRX    96.23 TRX
2026-03-29 09:00     Deposit       +50.00 TRX   146.23 TRX
2026-03-29 09:15     Order #1249   -5.40 TRX    140.83 TRX
```

每条记录对应 MERX 会计系统中的一条复式记账账本记录。所有记录的总和始终与你的当前余额一致——这是可验证和可审计的。

### 提现

如果你需要从 MERX 账户提取 TRX：

1. 导航到余额部分
2. 点击"提现"
3. 输入目标 TRON 地址
4. 输入提现金额
5. 确认提现

提现由 treasury-signer 服务处理并广播到 TRON 网络。处理时间取决于网络状况，但通常在几分钟内完成。

## 订单历史

订单历史视图提供所有过往订单的可搜索、可筛选列表。每条记录显示：

- **订单 ID**：用于参考的唯一标识符
- **日期/时间**：下单时间
- **能量数量**：购买了多少能量单位
- **时长**：租赁期限
- **供应商**：哪个供应商完成了订单
- **价格**：每单位价格（SUN）
- **总成本**：总扣费金额（TRX）
- **状态**：待处理、活跃、已完成或失败
- **目标地址**：能量委托到哪里

### 筛选和搜索

你可以按以下条件筛选订单：

- 日期范围
- 状态（活跃、已完成、全部）
- 供应商
- 目标地址

这对于为多个地址购买能量并需要按钱包或按应用追踪成本的组织特别有用。

## 生成 API 密钥

当你准备好以编程方式将 MERX 集成到应用中时，仪表板允许你无需联系支持即可生成 API 密钥。

1. 导航到设置或 API 部分
2. 点击"生成 API 密钥"
3. 为密钥提供标签（例如"生产服务器"、"测试环境"）
4. 密钥只显示一次——立即复制
5. 密钥以哈希形式存储在服务器上，无法再次检索

你可以管理多个 API 密钥，每个都有自己的标签。如果密钥泄露，从仪表板撤销它并生成新密钥。撤销密钥是即时的——使用被撤销密钥的所有请求将返回认证错误。

### API 密钥安全

API 密钥验证你对 MERX REST API、WebSocket 推送和 SDK 的请求。像对待密码一样对待它们：

- 永远不要将 API 密钥提交到版本控制
- 将它们存储在环境变量或密钥管理器中
- 为开发、测试和生产使用不同的密钥
- 定期轮换密钥

仪表板显示每个密钥的最后使用时间戳，便于识别和撤销未使用的密钥。

## 监控活跃委托

仪表板包含你活跃能量委托的实时视图：

```
Active Delegations
-------------------------------------------------
Target            Energy     Expires           Remaining
TAddr1...         65,000     2026-03-30 15:00  2h 30m
TAddr2...         200,000    2026-03-31 09:00  20h 30m
TAddr3...         500,000    2026-04-02 12:00  3d 2h 30m
```

对于每个活跃委托，你可以：

- 查看精确的到期时间
- 查看剩余时间
- 查看可用能量
- 通过为同一地址下新订单来延长委托

系统目前不支持提前取消活跃委托（TRON 上的能量委托是链上操作，运行到指定时长到期为止），但你可以通过为同一目标地址购买额外能量来延长现有委托。

## 能量估算工具

仪表板包含内置的能量估算工具。如果你不确定交易需要多少能量，可以直接从仪表板模拟：

1. 输入合约地址（例如 USDT 合约）
2. 选择函数（例如 transfer）
3. 输入参数（接收地址、金额）
4. 点击"估算"

该工具在底层调用 `triggerConstantContract`，返回你的特定交易在当前合约状态下所需的精确能量。这消除了猜测，防止能量购买过多或过少。

## 仪表板适用人群

### 企业运营者

如果你经营一家发送 USDT 付款的企业——工资、供应商付款、汇款——你不需要开发人员来集成 API。打开仪表板，充值 TRX，在每批转账前开始购买能量。成本节省是即时且显著的。

### 评估 MERX 的开发者

在进行 API 集成之前，使用仪表板测试服务。下几笔订单，观察定价，验证委托是否如预期到达链上。满意后，生成 API 密钥并转向程序化访问。

### 财务团队

订单历史和余额视图提供了财务团队所需的报告：花了多少钱、什么时候、用在什么上、来自哪个供应商。导出这些数据与你的内部会计系统对账。

### 偶尔使用者

如果你偶尔进行 TRON 交易——每周或每月几笔——仪表板可能就是你需要的全部。无需集成、无需代码、无需维护。只需登录，购买能量，节省 90% 的交易费用。

## 从仪表板到 API

仪表板和 API 共享同一个后端。你在仪表板中执行的每个操作——查看价格、下单、查看历史——都直接映射到一个 API 端点。当你准备好自动化：

```
Dashboard action          API equivalent
-----------------------------------------------------------
View prices               GET /api/v1/prices
Create order              POST /api/v1/orders
View order                GET /api/v1/orders/:id
View balance              GET /api/v1/balance
Estimate energy           POST /api/v1/estimate
```

从仪表板用户过渡到 API 用户是无缝的。你的账户、余额和订单历史完全延续。唯一的添加是你从仪表板本身生成的 API 密钥。
