# MERX vs CatFee vs ITRX vs TronSave: какой поставщик энергии дешевле?

Энергия TRON не бесплатна, но сколько вы платите, полностью зависит от того, где вы её покупаете. Разница между самым дешёвым и самым дорогим поставщиком за одно и то же количество энергии может составлять 3-4 раза. В этой статье сравниваются основные поставщики энергии TRON - CatFee, ITRX, TronSave, TEM, Netts и PowerSun - по ценообразованию, вариантам продолжительности и надёжности, а также объясняется, как агрегатор MERX избавляет от необходимости проверять каждого из них вручную.

## Рынок энергии TRON в 2026 году

Каждое взаимодействие со смарт-контрактом в TRON требует энергии. Отправка USDT стоит приблизительно 65 000 энергии. Своп на DEX может потребовать 200 000-500 000 энергии. Если у вас нет энергии, сеть TRON сжигает ваш TRX для покрытия стоимости - по ставке, примерно в 4 раза более дорогой, чем аренда энергии у стороннего поставщика.

Это создало рынок. Множество поставщиков теперь продают энергию, каждый с разными ценовыми моделями, тарифами по продолжительности и минимальными объёмами заказов. Некоторые предлагают аренду по фиксированной цене. Другие ведут одноранговые маркетплейсы, где цена колеблется в зависимости от спроса и предложения. Некоторые специализируются на краткосрочной аренде для отдельных транзакций. Другие нацелены на долгосрочные контракты для бизнеса с устойчивым объёмом.

Проблема в том, что ни один поставщик не является всегда самым дешёвым. Цены меняются в течение дня в зависимости от сетевого спроса, запасов поставщика и конкурентной динамики. То, что было самым дешёвым час назад, может не быть самым дешёвым сейчас.

## Профили поставщиков

### CatFee

CatFee предлагает аренду энергии по фиксированным ценам с акцентом на простоту. Их основной продукт - энергия на 1 час, что покрывает подавляющее большинство сценариев использования: большинству пользователей энергия нужна для одной транзакции, и 1 час предоставляет достаточно времени для её выполнения.

**Ключевые характеристики:**
- Фиксированные цены, обновляемые периодически
- Основная продолжительность: 1 час
- Конкурентоспособен для краткосрочного одноразового использования
- Простой API с понятной интеграцией
- Популярен среди индивидуальных пользователей и небольших приложений

CatFee заработал репутацию надёжного поставщика в сегменте краткосрочной аренды. Их часовой продукт хорошо подходит для пользователей, которым нужна энергия для отдельного перевода USDT или взаимодействия с контрактом и которые не хотят думать о выборе продолжительности.

### ITRX

ITRX предоставляет энергию по четырём отдельным тарифным уровням продолжительности, что делает его одним из наиболее гибких поставщиков с фиксированными ценами. Их ступенчатая модель позволяет пользователям выбирать период аренды, соответствующий их фактическому паттерну использования.

**Уровни продолжительности:**
- 1 час - для отдельных транзакций
- 1 день (24 часа) - для пакетных операций
- 3 дня - для среднесрочных проектов
- 30 дней - для устойчивых операционных потребностей

**Ключевые характеристики:**
- Фиксированные цены по уровням, регулярно обновляемые
- Конкурентное ценообразование, особенно на длительных сроках
- Хорошо документированный API
- Надёжное исполнение заказов стандартных размеров

ITRX - хороший выбор для пользователей, которые знают свои потребности в энергии заранее и хотят предсказуемое ценообразование. Уровень на 30 дней особенно полезен для компаний, ведущих непрерывные операции в TRON.

### TronSave

TronSave работает по модели однорангового маркетплейса, а не продаёт энергию по фиксированным ценам. Делегаторы размещают свой застейканный TRX для аренды, а покупатели делают ставки на доступное предложение. Это создаёт динамичную ценовую среду, где стоимость колеблется в зависимости от рыночных условий.

**Ключевые характеристики:**
- Одноранговый маркетплейс с динамическим ценообразованием
- Гибкие варианты продолжительности
- Цены могут быть ниже, чем у поставщиков с фиксированными ценами, в периоды низкого спроса
- Цены могут вырастать в периоды высокого спроса
- Более крупные минимальные объёмы заказов для некоторых предложений
- Устоявшаяся платформа со значительной ликвидностью

TronSave обычно предлагает лучшие цены, когда сетевой спрос низок и предложение делегаторов велико. Однако в пиковые периоды цены могут превышать цены поставщиков с фиксированными ценами. Он вознаграждает пользователей, которые могут выбирать время покупки.

### TEM (Tron Energy Market)

TEM - ещё один одноранговый маркетплейс энергии, ориентированный на крупные заказы. Он обслуживает в первую очередь крупных пользователей - биржи, платёжные процессоры и крупные dApp - которым нужны значительные объёмы энергии.

**Ключевые характеристики:**
- Модель однорангового маркетплейса
- Оптимизирован для крупных заказов
- Конкурентное ценообразование на больших объёмах
- Менее подходит для небольших, индивидуальных транзакций
- Растущая ликвидность в сегменте крупных заказов

TEM - не лучший выбор для того, кто отправляет один перевод USDT. Но для платёжного процессора, выполняющего тысячи транзакций в день, оптовое ценообразование TEM может обеспечить значительную экономию.

### Netts

Netts предлагает энергию по фиксированным ценам с фокусом на короткие сроки и агрессивное ценообразование. Они часто занимают одну из самых низких позиций по цене на энергию на 1 час, что делает их сильным вариантом для пользователей, чувствительных к цене.

**Ключевые характеристики:**
- Фиксированные цены, часто самые низкие на рынке
- Фокус на короткие сроки (1 час, 1 день)
- Быстрое делегирование (энергия доставляется быстро после заказа)
- Низкие минимальные объёмы заказов
- Хорошая надёжность API

Netts конкурирует прежде всего по цене. Для пользователей, чей единственный приоритет - платить минимум за единицу энергии для одной транзакции, Netts часто является ответом, хотя это варьируется по мере того, как поставщики корректируют свои ставки.

### PowerSun

PowerSun использует другой подход с 10 отдельными уровнями продолжительности, предлагая самый широкий диапазон периодов аренды на рынке. Такая гранулярность позволяет пользователям точно подбирать аренду энергии под свой операционный график.

**Уровни продолжительности:**
- 10 минут, 30 минут, 1 час, 3 часа, 6 часов
- 12 часов, 1 день, 3 дня, 7 дней, 30 дней

**Ключевые характеристики:**
- 10 уровней продолжительности - больше всех на рынке
- Фиксированные цены по уровням
- Особенно силён в субчасовых арендах (10 и 30 минут)
- Хорош для пользователей, которым нужна энергия на очень короткие окна
- Проверенный технологический стек

Уровни на 10 и 30 минут PowerSun заполняют пробел, который большинство других поставщиков не закрывают. Если вам нужна энергия ровно для одной транзакции и вы хотите минимизировать окно аренды, эти сверхкороткие сроки могут быть экономически эффективнее, чем оплата полного часа.

## Сравнение цен

Цены на энергию указываются в SUN за единицу энергии в день. Чем ниже, тем лучше. Приведённая ниже таблица отражает репрезентативные рыночные цены. Фактические цены меняются непрерывно.

| Поставщик | 1 час | 1 день | 3 дня | 7 дней | 30 дней |
|---|---|---|---|---|---|
| CatFee | 28-35 SUN | - | - | - | - |
| ITRX | 30-38 SUN | 26-32 SUN | 24-30 SUN | - | 22-28 SUN |
| TronSave | 25-45 SUN | 24-40 SUN | 23-38 SUN | 22-35 SUN | 20-32 SUN |
| TEM | 30-42 SUN | 26-36 SUN | 24-34 SUN | 22-32 SUN | 20-30 SUN |
| Netts | 22-32 SUN | 24-30 SUN | - | - | - |
| PowerSun | 26-34 SUN | 24-32 SUN | 22-30 SUN | 22-28 SUN | 20-26 SUN |

**Ключевые наблюдения:**

- Netts часто предлагает самую низкую цену на 1 час, но доступность может варьироваться
- Диапазон TronSave самый широкий, поскольку одноранговое ценообразование колеблется в зависимости от спроса и предложения
- Более длительные сроки, как правило, стоят дешевле за день у всех поставщиков
- Ни один поставщик не доминирует по всем срокам
- Разница между самым дешёвым и самым дорогим вариантом для одного и того же срока может составлять 60-80%

## Сравнение по продолжительности: какой поставщик для какого сценария

Разные паттерны использования требуют разных поставщиков.

### Один перевод USDT (нужна энергия для 1 транзакции)

**Лучшие варианты:** Netts (обычно самый дешёвый), CatFee (самый надёжный), PowerSun с уровнем 10 минут (самая низкая общая стоимость при наличии)

Для разового перевода вам нужна энергия на минуты, а не на часы. Уровни на 10 или 30 минут от PowerSun могут оказаться дешевле в абсолютном выражении, чем аренда на 1 час у другого поставщика, даже если ставка за день выше. Вы платите за меньшее время.

### Пакетная обработка (50-200 транзакций за несколько часов)

**Лучшие варианты:** ITRX уровень на 1 день, PowerSun уровень на 3 или 6 часов

Пакетные операции требуют устойчивой энергии в определённом окне. Аренда на 1 день даёт комфортный запас без переплаты за многодневное обязательство.

### Непрерывные операции (биржа, платёжный процессор)

**Лучшие варианты:** PowerSun на 30 дней, ITRX на 30 дней, TronSave (если вы можете выбирать время покупки в периоды низкого спроса)

Долгосрочная аренда предлагает самые низкие ставки за день. Для непрерывных операций уровень на 30 дней от любого поставщика будет значительно дешевле, чем многократная покупка энергии на 1 час.

### Непредсказуемый, пиковый трафик

**Лучшие варианты:** Агрегатор MERX с автоматической маршрутизацией

Если ваш объём транзакций непредсказуем - иногда 10 транзакций в час, иногда 1 000 - ни один конкретный поставщик или уровень продолжительности не является оптимальным. Именно здесь агрегатор приносит наибольшую пользу.

## Преимущество агрегатора

Проверить одного поставщика и предположить, что вы получили лучшую цену, - дорогостоящая ошибка. Вот почему.

Рассмотрим пользователя, которому нужно 65 000 энергии для перевода USDT. На момент запроса:

- Netts котирует 24 SUN/энергию/день
- CatFee котирует 32 SUN/энергию/день
- ITRX котирует 30 SUN/энергию/день
- TronSave имеет предложения по 26-40 SUN/энергию/день

Если пользователь проверяет только CatFee, он платит 32 SUN. Если проверяет Netts - платит 24 SUN. Это разница в 33% за совершенно одинаковую энергию в одной и той же сети.

Теперь умножьте это на тысячи транзакций. Платёжный процессор, выполняющий 1 000 переводов USDT в день, значительно сэкономит, постоянно выбирая самого дешёвого поставщика, но вручную сравнивать шесть поставщиков перед каждой транзакцией непрактично.

Это фундаментальная проблема, которую решает агрегатор. Он проверяет всех поставщиков в реальном времени, каждый раз, и автоматически направляет к самому дешёвому варианту.

## Как MERX решает проблему сравнения

MERX - это биржа ресурсов, которая агрегирует энергию (и bandwidth) от множества поставщиков. Когда вы размещаете заказ через MERX, платформа:

1. Одновременно опрашивает всех активных поставщиков
2. Сравнивает цены для запрошенного объёма энергии и продолжительности
3. Направляет заказ к поставщику с самой низкой ценой
4. Обрабатывает делегирование и подтверждение
5. Переключается на следующего по дешевизне поставщика, если основной недоступен

Пользователь видит единый интерфейс и единую цену - всегда лучшую из доступных на данный момент. Сложность управления аккаунтами у нескольких поставщиков, сравнения API и обработки переключений абстрагирована.

```bash
# Установка JavaScript SDK MERX
npm install merx-sdk
```

```typescript
import { MerxClient } from 'merx-sdk';

const merx = new MerxClient({ apiKey: 'your-api-key' });

// Получить лучшую цену среди всех поставщиков
const best = await merx.getBestPrice({
  energy: 65000,
  duration: '1h'
});

console.log(best.provider);   // например, "netts"
console.log(best.price);      // например, 24 SUN
console.log(best.total_trx);  // общая стоимость в TRX

// Разместить заказ - MERX направляет к самому дешёвому поставщику
const order = await merx.createOrder({
  energy: 65000,
  duration: '1h',
  target: 'TJnVmb5rFLHPqfDMRMGwMH2iofhzN3KXLG'
});
```

Python SDK предоставляет ту же функциональность:

```bash
pip install merx-sdk
```

```python
from merx_sdk import MerxClient

merx = MerxClient(api_key="your-api-key")
```

## Пример расчёта: 1 000 переводов USDT

Рассчитаем стоимость выполнения 1 000 переводов USDT при разных подходах. Каждый перевод требует приблизительно 65 000 энергии.

### Без аренды энергии (сжигание TRX)

Когда у вас нет энергии, TRON сжигает TRX с вашего баланса для покрытия стоимости. При текущих параметрах сети стандартный перевод USDT сжигает приблизительно 13,5 TRX.

- Стоимость за перевод: ~13,5 TRX
- Стоимость за 1 000 переводов: ~13 500 TRX
- При TRX = $0,24: ~$3 240

### CatFee (1 час, фиксированная цена 32 SUN)

- Стоимость энергии за перевод: ~2,08 TRX
- Bandwidth и комиссии: ~0,5 TRX
- Стоимость за перевод: ~2,58 TRX
- Стоимость за 1 000 переводов: ~2 580 TRX
- При TRX = $0,24: ~$619

### Netts (1 час, фиксированная цена 24 SUN)

- Стоимость энергии за перевод: ~1,56 TRX
- Bandwidth и комиссии: ~0,5 TRX
- Стоимость за перевод: ~2,06 TRX
- Стоимость за 1 000 переводов: ~2 060 TRX
- При TRX = $0,24: ~$494

### ITRX (30 дней по 22 SUN, при готовности взять обязательство)

- Стоимость энергии за перевод: ~1,43 TRX (амортизированная)
- Bandwidth и комиссии: ~0,5 TRX
- Стоимость за перевод: ~1,93 TRX
- Стоимость за 1 000 переводов: ~1 930 TRX
- При TRX = $0,24: ~$463

### MERX (автоматическая маршрутизация к самому дешёвому поставщику)

MERX проверяет всех поставщиков в момент каждого заказа и направляет к самому дешёвому доступному варианту. Предполагая, что MERX захватывает лучшую доступную цену на каждой транзакции (которая варьируется между поставщиками в течение дня):

- Средняя стоимость энергии за перевод: ~1,50 TRX (средневзвешенная самого дешёвого на момент запроса)
- Bandwidth и комиссии: ~0,5 TRX
- Стоимость за перевод: ~2,00 TRX
- Стоимость за 1 000 переводов: ~2 000 TRX
- При TRX = $0,24: ~$480

**Итог экономии за 1 000 переводов:**

| Метод | Общая стоимость (TRX) | Общая стоимость (USD) | Экономия vs сжигание |
|---|---|---|---|
| Без энергии (сжигание TRX) | 13 500 | $3 240 | - |
| CatFee (32 SUN) | 2 580 | $619 | 81% |
| Netts (24 SUN) | 2 060 | $494 | 85% |
| ITRX 30 дней (22 SUN) | 1 930 | $463 | 86% |
| MERX (автомаршрутизация) | 2 000 | $480 | 85% |

Ключевой вывод не в том, что MERX всегда дешевле каждого отдельного поставщика на каждой отдельной транзакции - иногда Netts будет самым дешёвым, иногда ITRX. Ценность в том, что MERX стабильно находится вблизи нижней границы цен, потому что всегда выбирает лучший текущий вариант, без необходимости поддерживать аккаунты у шести поставщиков и сравнивать цены вручную.

## Когда использовать одного поставщика, а когда агрегатор

**Используйте одного поставщика, когда:**
- Вы договорились об индивидуальной ставке с этим поставщиком
- У вас есть долгосрочный контракт с зафиксированной ценой
- Вам нужна энергия только для нескольких транзакций в месяц
- Вы протестировали конкретного поставщика и ему доверяете

**Используйте агрегатор, такой как MERX, когда:**
- Вы выполняете более 100 транзакций в день
- Ваш объём транзакций непредсказуем
- Вы хотите минимизировать постоянные операционные затраты
- Вам нужно автоматическое переключение при отказе поставщика
- Вы хотите единый API и единые расчётные отношения

Для программного доступа [документация MERX](https://merx.exchange/docs) охватывает аутентификацию API, создание заказов и ценовые запросы. SDK доступны для JavaScript и Python. Для интеграции AI-агентов [MCP-сервер MERX](https://github.com/Hovsteder/merx-mcp) предоставляет 52 инструмента, доступных через Model Context Protocol.

Рынок энергии TRON конкурентен, и эта конкуренция выгодна покупателям. Но для получения этих преимуществ необходимо видеть весь рынок, а не только прайс-лист одного поставщика. Для этого и существует агрегатор.
