# Торговля без комиссий: бизнес-модель MERX

MERX не взимает комиссию за торговлю энергией. Никакой надбавки к ценам поставщиков. Никаких скрытых платежей. Никакого спреда. Вы платите ровно столько, сколько взимает поставщик, и MERX не добавляет ничего сверху.

Естественно, это вызывает вопросы. Как бизнес может существовать без доходов? Это потерпевшая убытки стратегия, которая закончится резким повышением цен? В чем подвох?

Подвоха нет. Но есть стратегия. В этой статье объясняется, как работает модель нулевой комиссии MERX, почему она существует и как платформа планирует развиваться и получать доход со временем.

---

## Что именно означает нулевая комиссия

Когда вы покупаете энергию через MERX, цена, которую вы платите, равна оптовой цене поставщика. Если самый дешевый поставщик предлагает энергию по 85 SUN за единицу, вы платите 85 SUN за единицу. MERX не добавляет никакую маржу.

Для контекста вот как выглядит типичная покупка энергии:

```
Заказано энергии:         65 000 единиц
Лучшая цена поставщика:   85 SUN/единица
Общая стоимость:          5 525 000 SUN = 5.525 TRX

Надбавка MERX:            0 SUN
Комиссия MERX:            0 SUN
Итого к оплате:           5 525 000 SUN = 5.525 TRX
```

Это можно проверить. Каждый ответ заказа содержит имя поставщика и цену. Вы можете проверить собственный API поставщика, чтобы подтвердить, что MERX не завышает цену. Прозрачность намеренна — она создает доверие и делает утверждение о нулевой комиссии проверяемым.

---

## Почему нулевая комиссия

### Фаза захвата рынка

MERX находится на фазе захвата рынка. Рынок агрегирования энергии на TRON в зачаточном состоянии — большинство разработчиков и компаний по-прежнему интегрируются напрямую с отдельными поставщиками или, еще хуже, сжигают TRX на каждую транзакцию. Приоритет сейчас не доход, а принятие платформы.

Нулевая комиссия устраняет главное возражение против использования агрегатора: "Зачем мне платить посреднику, если я могу обратиться напрямую?" С нулевой комиссией посредник добавляет ценность (маршрутизация по лучшей цене, отказоустойчивость, единый API) без добавления стоимости.

### Эффект маховика

Каждый новый пользователь на MERX генерирует данные: объемы заказов, чувствительность к цене, предпочтения поставщиков, паттерны использования. Эти данные улучшают платформу:

- **Больший объем заказов** дает MERX рычаги для согласования лучших ставок с поставщиками.
- **Лучшие ставки** привлекают больше пользователей.
- **Больше пользователей** генерируют больше данных для оптимизации маршрутизации.
- **Лучшая маршрутизация** приводит к лучшим ценам и надежности.

Нулевая комиссия ускоряет маховик. Это инвестиция в сетевые эффекты, которые растут со временем.

### Сравнение с прямыми ценами поставщиков

Отдельные поставщики устанавливают свои собственные цены и уже включают свою маржу. Когда вы покупаете у TronSave по 88 SUN за единицу, эти 88 включают операционные расходы и прибыль TronSave. MERX передает эту цену напрямую, не добавляя к ней ничего.

Некоторые поставщики предлагают скидки за объем крупным покупателям. MERX, агрегируя объемы многих покупателей, может потенциально получить доступ к этим скидкам и передать экономию отдельным пользователям, которые не квалифицировались бы самостоятельно. Это будущее преимущество, которое растет с объемом платформы.

## Как MERX поддерживает операции сегодня

Запуск MERX не бесплатен. Платформе требуются серверы, разработка, мониторинг и операционная поддержка. На фазе нулевой комиссии эти расходы финансируются учредительской командой как бизнес-инвестиция.

### Текущая структура расходов

```
Инфраструктура:
  - Выделенный сервер (Hetzner):           ~$150/месяц
  - Домен и SSL:                            ~$20/месяц
  - Мониторинг и оповещение:                ~$50/месяц

Разработка:
  - Время учредительской команды:          Не финансируется извне
  - Нет венчурного капитала (пока)
  - Нет продажи токенов

Операции:
  - Доступ к API поставщиков:               Бесплатный (поставщики хотят объема)
  - Доступ к узлу TRON:                     Публичные узлы + собственный узел
  - Redis, PostgreSQL:                      Самостоятельно на выделенном сервере
```

Расходы на инфраструктуру скромны по любым стандартам. Это тощая операция по замыслу — каждый сэкономленный доллар на инфраструктуре — это доллар, который не нужно возвращать пользователям.

## Модель доходов будущего

Нулевая комиссия — не постоянное состояние. Это входная цена для рынка, который MERX намерен развивать. Вот как модель доходов развивается:

### Этап 1: Нулевая комиссия (текущий)

- 0% комиссия на все сделки.
- Цель: привлечение пользователей, отношения с поставщиками, зрелость платформы.
- Продолжительность: до достижения значительного объема заказов.

### Этап 2: Премиум-функции

Первые доходы поступают от добавленных услуг, которые выходят за рамки базовой маршрутизации цен:

**Стоящие заказы и автоматизация**

```
Базовый (бесплатный):  Ручные заказы, маршрутизация по лучшей цене
Премиум:                Стоящие заказы, автообновление, оповещения о цене
                        Запланированная закупка энергии
                        Webhook-уведомления для событий делегирования
```

**Аналитика и инсайты**

```
Базовый (бесплатный):  Текущие цены, базовая история заказов
Премиум:                Модели прогноза цен
                        Оценка надежности поставщика
                        Рекомендации по оптимизации затрат
                        Пользовательские отчеты и экспорт
```

**Приоритетная поддержка**

```
Базовый (бесплатный):  Документация, поддержка сообщества
Премиум:                Прямой канал поддержки
                        Гарантии SLA на выполнение заказов
                        Выделенное управление аккаунтом
```

Базовая услуга — маршрутизация заказов по лучшей цене — остается бесплатной. Премиум-функции предназначены для опытных пользователей, которые получают достаточно ценности от платформы, чтобы оправдать подписку.

### Этап 3: Ценообразование на основе объема

Для пользователей с очень большим объемом (миллионы единиц энергии в день) MERX может ввести небольшую комиссию, отражающую операционные расходы на обработку крупных заказов:

```
Уровень объема:        Комиссия:
0 - 1М энергии/месяц:  0% (всегда бесплатно)
1М - 10М:               0.5%
10М - 100М:             0.3%
100М+:                  По договору
```

Даже при 0.5% общая стоимость все еще ниже того, что большинство пользователей платят при прямой интеграции с поставщиком (где они не могут получить маршрутизацию по лучшей цене).

### Этап 4: Услуги для поставщиков

По мере роста MERX как доминирующего слоя агрегирования поставщики получают выгоду от потока заказов. Будущие источники доходов могут включать:

- **Приоритетное размещение**: поставщики платят за приоритет в маршрутизации (с прозрачностью — покупатели всегда видят фактическую цену).
- **Аналитика для поставщиков**: данные о доле рынка, конкурентная информация о ценах.
- **Услуги расчетов**: MERX обрабатывает сбор платежей и переводы, взимая с поставщиков небольшую комиссию за обработку.

## Сравнение с надбавками поставщиков

Как нулевая комиссия MERX сравнивается с тем, что уже взимают поставщики?

Поставщики — не благотворительные организации. Их котируемые цены включают их операционные расходы и маржу прибыли. Типичная структура надбавки поставщика выглядит так:

```
Структура затрат поставщика:
  - Стоимость стейкинга TRX (альтернативная стоимость):  Базовая стоимость
  - Инфраструктура (серверы, узлы):                     5-10% от базовой
  - Разработка и техническое обслуживание:              5-10% от базовой
  - Маржа прибыли:                                       10-30% от базовой

Предполагаемая надбавка к сырой стоимости стейкинга:    20-50%
```

Когда вы покупаете по 85 SUN за единицу у поставщика, примерно 55-70 SUN покрывают сырую стоимость стейкинга, а 15-30 SUN — это накладные расходы и маржа поставщика. Это нормально и устойчиво.

MERX не добавляет еще один слой маржи. 85 SUN, которые взимает поставщик — это 85 SUN, которые вы платите. Сравните с другими моделями агрегирования:

```
Традиционный агрегатор:
  Цена поставщика:      85 SUN/единица
  Надбавка агрегатора:  5-15%
  Вы платите:           89-98 SUN/единица

MERX:
  Цена поставщика:      85 SUN/единица
  Надбавка MERX:        0%
  Вы платите:           85 SUN/единица
```

## Вопрос доверия

"Если ты ничего не берешь, то товар — это ты." Это разумное беспокойство в отношении любой бесплатной услуги. Давайте обратимся к нему напрямую.

### MERX не продает ваши данные

Данные заказов используются для оптимизации маршрутизации и улучшения платформы. Они не продаются третьим сторонам. Нет модели рекламы. Объемы и паттерны ваших транзакций — это ваше дело.

### MERX не хранит ваши средства в опеке

MERX держит остатки депозитов для выполнения заказов. Это операционные остатки, а не опекунские активы. Вы можете в любое время вывести свой баланс. Платформа не инвестирует, не ссужает или иным образом не использует ваши депонированные средства.

### MERX не совершает фронтранинг заказов

Исполнитель заказа маршрутизирует по лучшей доступной цене в момент выполнения. Он не задерживает заказы, ожидая лучших цен (что принесло бы выгоду MERX, если бы она держала позицию) и не маршрутизирует к более дорогим поставщикам, когда доступны более дешевые.

### Код можно проверить

SDK MERX имеют открытый исходный код:

- Python SDK: [https://github.com/Hovsteder/merx-sdk-python](https://github.com/Hovsteder/merx-sdk-python)
- MCP Server: [https://github.com/Hovsteder/merx-mcp](https://github.com/Hovsteder/merx-mcp)

Каждый заказ включает имя поставщика и хеш транзакции в блокчейне. Вы можете независимо проверить, что делегирование произошло по цене, указанной MERX.

## Почему не использовать поставщиков напрямую?

Если MERX не взимает комиссию, почему не интегрироваться с поставщиками напрямую и пропустить слой агрегирования?

Можете. Но вот что вы теряете:

### Маршрутизация по лучшей цене

Цены поставщиков меняются в течение дня. Самый дешевый поставщик в 9 утра может быть самым дорогим в 3 часа дня. Без непрерывного мониторинга (что MERX делает каждые 30 секунд) вы, вероятно, не получите лучшую цену ни в какой момент.

### Автоматическая отказоустойчивость

Если ваш единственный поставщик выходит из строя, ваше приложение перестает работать. С MERX отказ поставщика невидим для вас — заказы автоматически маршрутизируются к следующему самому дешевому варианту.

### Операционная простота

Семь интеграций с поставщиками означают семь наборов документации API, семь методов аутентификации, семь форматов ошибок, семь наборов изменений API для отслеживания. Одна интеграция с MERX заменяет все это.

### Защита от будущего

На рынок входят новые поставщики. Существующие поставщики меняют свои API или закрываются. С MERX новые поставщики автоматически доступны вам, а несуществующие поставщики удаляются — изменения кода не требуются.

Модель нулевой комиссии означает, что вы получаете все эти преимущества без дополнительных затрат по сравнению с прямой интеграцией. Единственная рациональная причина идти напрямую — если у вас есть специфические требования, которые MERX не поддерживает — и если это так, команда хочет об этом узнать.

## Для ранних принявших

Ставка нулевой комиссии для ранних пользователей не ограничена по времени. Пользователи, которые присоединятся на этой фазе, будут закрепить благоприятные условия по мере развития платформы. Это не приманка и подмена; это признание того, что ранние пользователи берут на себя больше риска (более новая платформа, меньше истории) и заслуживают соответствующей награды.

Если модель доходов со временем будет включать комиссии, ранние пользователи либо:
- Сохранят нулевую комиссию навсегда, либо
- Получат значительные скидки по сравнению с более поздними пользователями

Детали будут сообщены задолго до любых изменений в ценообразовании.

## Начало работы

Создайте аккаунт и начните торговать без комиссии:

1. Посетите [https://merx.exchange](https://merx.exchange)
2. Создайте аккаунт
3. Создайте ключ API
4. Сделайте ваш первый заказ

```typescript
import { MerxClient } from 'merx-sdk';

const client = new MerxClient({ apiKey: 'your-key' });

// Смотрите текущие лучшие цены (без добавления комиссии)
const prices = await client.getPrices({ energy: 65000 });
console.log(`Best price: ${prices.bestPrice.perUnit} SUN/unit`);
console.log(`Provider: ${prices.bestPrice.provider}`);
console.log(`MERX fee: 0`);
```

Документация: [https://merx.exchange/docs](https://merx.exchange/docs)
