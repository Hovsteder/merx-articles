# Представляем MERX: первую биржу ресурсов TRON

MERX - первый агрегатор-биржа ресурсов сети TRON - энергии и bandwidth - созданный для решения проблемы фрагментированного рынка поставщиков, который вынуждает бизнес переплачивать за каждый перевод USDT в блокчейне TRON. Объединяя множество поставщиков энергии на одной платформе со сравнением цен в реальном времени и автоматической маршрутизацией к лучшей цене, MERX снижает стоимость переводов USDT до 94 процентов по сравнению с механизмом сжигания TRX по умолчанию.

## Какую проблему решает MERX

Каждый перевод USDT в сети TRON требует сетевого ресурса - энергии. Без него протокол сжигает TRX с вашего кошелька - примерно 27 TRX за стандартный перевод. Существует целый рынок поставщиков аренды энергии, предлагающих делегированную энергию за малую долю стоимости сжигания.

Но этот рынок фрагментирован. Существует минимум семь крупных поставщиков, каждый со своим API, ценовой моделью, системой аутентификации и паттернами доступности. Цены варьируются от 22 SUN до 80 SUN за единицу энергии в любую конкретную минуту - разброс более чем в 3,6 раза между самым дешёвым и самым дорогим вариантом.

Для разработчика, создающего платёжную систему, биржу или любое приложение, программно отправляющее USDT, эта фрагментация создаёт реальные инженерные проблемы:

**Множественные интеграции.** У каждого поставщика свой SDK, формат API и обработка ошибок. Интеграция со всеми для получения лучшей цены означает поддержку нескольких кодовых баз.

**Отсутствие прозрачности цен.** Нет центральной книги заявок или ценового потока. Чтобы узнать лучшую цену, нужно опросить каждого поставщика по отдельности и сравнить.

**Отсутствие резервного переключения.** Если ваш единственный поставщик выходит из строя или исчерпывает мощности, ваши транзакции либо не проходят, либо откатываются к дорогостоящему сжиганию TRX.

**Ручной мониторинг цен.** Цены меняются в течение дня. Самый дешёвый поставщик в 9 утра может стать самым дорогим к 15:00. Без непрерывного мониторинга вы не можете гарантировать, что получаете лучшую сделку.

MERX устраняет все эти проблемы с помощью единой точки интеграции.

## Как работает MERX

Архитектура построена вокруг трёх основных операций, работающих непрерывно.

### Опрос цен

MERX подключается к каждому крупному поставщику энергии TRON и опрашивает их текущие цены каждые 30 секунд. Это создаёт индекс цен в реальном времени по всему рынку. Когда вы проверяете цены на MERX, вы видите текущую ставку каждого поставщика и кто из них самый дешёвый в данный момент.

Опрос - это не просто проверка цены. MERX также верифицирует доступность, мощности и поддерживаемые диапазоны продолжительности поставщика. У поставщика может быть отличная цена, но нет мощности для выполнения вашего заказа. MERX отфильтровывает таких перед тем, как представить варианты.

### Маршрутизация к лучшей цене

Когда вы размещаете заказ на энергию через MERX, платформа не просто перенаправляет его поставщику по умолчанию. Она оценивает всех подключённых поставщиков по параметрам вашего конкретного заказа - количество, продолжительность, целевой адрес - и направляет к самому дешёвому поставщику, способному выполнить заказ.

Если самый дешёвый поставщик не может выполнить заказ (сетевые проблемы, исчерпание мощности, тайм-аут), MERX автоматически переключается на следующий по дешевизне вариант. Это переключение прозрачно. Вы получаете энергию в любом случае; платформа берёт на себя сложность маршрутизации.

### Исполнение заказов

После выбора поставщика MERX выполняет делегирование энергии в блокчейне. Энергия появляется на вашем целевом адресе, готовая к использованию. Весь процесс - от вызова API до делегирования энергии - обычно завершается за секунды.

## Компоненты платформы

MERX - это не просто веб-интерфейс. Это полноценная платформа с несколькими путями интеграции, разработанными для различных сценариев использования.

### Веб-биржа

Основной интерфейс на [merx.exchange](https://merx.exchange) предоставляет обзор рынка энергии в реальном времени. Вы можете видеть текущие цены всех поставщиков, размещать заказы вручную, проверять баланс и просматривать историю заказов. Интерфейс спроектирован для профессионалов: тёмная тема, никакого визуального шума, информационно насыщенные макеты.

### REST API

API предоставляет 46 конечных точек, покрывающих полный цикл торговли энергией:

- **Рыночные данные** - текущие цены, история цен, сравнение поставщиков
- **Заказы** - создание, отслеживание, просмотр заказов на энергию и bandwidth
- **Регулярные заказы** - повторяющиеся покупки энергии по расписанию
- **Управление аккаунтом** - балансы, депозиты, выводы
- **Мониторинг** - настройка мониторов ресурсов и пороговых уведомлений
- **Утилиты для адресов** - валидация адресов, проверка ресурсов в блокчейне

Все конечные точки версионированы под `/api/v1/` и возвращают стандартизированные ответы об ошибках. POST-конечные точки поддерживают ключи идемпотентности для предотвращения дублирования заказов.

Полная документация доступна на [merx.exchange/docs](https://merx.exchange/docs), включая 36 страниц справочников по конечным точкам, руководства по аутентификации, таблицы кодов ошибок и примеры интеграции.

### WebSocket

Для приложений, которым нужны обновления цен в реальном времени без опроса, MERX предоставляет соединение WebSocket. Подпишитесь на ценовые каналы и получайте обновления по мере их появления - каждые 30 секунд при изменении цен.

### JavaScript SDK

Официальный SDK для JavaScript/TypeScript оборачивает REST API в типизированный клиент со встроенной обработкой ошибок, логикой повторных попыток и вспомогательными методами.

```javascript
import { MerxClient } from 'merx-sdk';

const merx = new MerxClient({
  apiKey: process.env.MERX_API_KEY
});

// Получить текущую лучшую цену на энергию
const prices = await merx.getPrices();
console.log('Best price:', prices.energy.best.price, 'SUN/unit');
console.log('Provider:', prices.energy.best.provider);

// Сравнить всех поставщиков
const comparison = await merx.compareProviders();
for (const provider of comparison) {
  console.log(`${provider.name}: ${provider.price} SUN`);
}

// Купить энергию по лучшей цене
const order = await merx.createOrder({
  resourceType: 'ENERGY',
  amount: 65000,
  duration: '1h',
  targetAddress: 'TRecipientAddressHere'
});

console.log('Cost:', order.totalCost, 'SUN');
```

Доступен на [npm](https://www.npmjs.com/package/merx-sdk) как `merx-sdk`. Исходный код на [GitHub](https://github.com/Hovsteder/merx-sdk-js).

### Python SDK

Python SDK предоставляет ту же функциональность для Python-приложений с поддержкой синхронного и асинхронного режимов.

```python
from merx_sdk import MerxClient

client = MerxClient(api_key="YOUR_API_KEY")

# Получить текущую лучшую цену на энергию
prices = client.get_prices()
print(f"Best price: {prices['energy']['best']['price']} SUN/unit")

# Рассчитать потенциальную экономию
savings = client.calculate_savings(
    energy_amount=65000,
    num_transfers=1000
)
print(f"Monthly savings: {savings['savings_trx']} TRX")

# Разместить заказ
order = client.create_order(
    resource_type="ENERGY",
    amount=65000,
    duration="1h",
    target_address="TRecipientAddressHere"
)
print(f"Order {order['id']}: {order['status']}")
```

Доступен на [PyPI](https://pypi.org/project/merx-sdk/) как `merx-sdk`. Исходный код на [GitHub](https://github.com/Hovsteder/merx-sdk-python).

### MCP-сервер для AI-агентов

MERX включает сервер Model Context Protocol (MCP), который позволяет AI-агентам и приложениям на базе LLM взаимодействовать с рынком энергии TRON напрямую. AI-агент может проверять цены, размещать заказы, мониторить адреса и управлять покупками энергии через стандартные вызовы инструментов.

Это особенно актуально, поскольку AI-агенты всё активнее управляют операциями в блокчейне. AI-агент, управляющий казначейством, обрабатывающий платежи или управляющий DeFi-стратегией, может использовать MCP-сервер MERX для оптимизации затрат на энергию без написания специального интеграционного кода.

Доступен на [npm](https://www.npmjs.com/package/merx-mcp) как `merx-mcp`. Исходный код на [GitHub](https://github.com/Hovsteder/merx-mcp).

## Чем MERX отличается от единичных поставщиков

Единичный поставщик энергии - это вендор. MERX - это маркетплейс. Это различие важно по нескольким конкретным причинам.

**Цена.** Единичный поставщик устанавливает собственную цену. MERX показывает вам цену каждого поставщика и направляет к самому дешёвому. В любую конкретную минуту самый дешёвый поставщик отличается. За месяц экономия от постоянного попадания на лучшую цену суммируется в значительные суммы.

**Доступность.** Если единичный поставщик исчерпывает мощности или уходит в офлайн, ваша покупка энергии не удаётся. MERX автоматически переключается на следующего доступного поставщика. Вашему приложению не нужно обрабатывать логику переключения.

**Прозрачность.** С единичным поставщиком у вас нет видимости того, получаете ли вы конкурентную цену. MERX показывает полный рыночный спред, чтобы вы могли точно видеть, куда был направлен ваш заказ и почему.

**Простота интеграции.** Интеграция с 7 поставщиками означает поддержку 7 API-интеграций, 7 процессов аутентификации и 7 наборов обработки ошибок. Интеграция с MERX означает один API, один SDK, один набор учётных данных.

**Нейтральность.** MERX не управляет собственной операцией по стейкингу энергии, конкурирующей с поставщиками на платформе. Это чистый агрегатор, заинтересованный в маршрутизации к лучшей цене, а не в продвижении собственного предложения.

## Реальные результаты в блокчейне

MERX выполнил заказы на энергию в основной сети TRON с верифицированными результатами. Восемь транзакций подтверждены в блокчейне, демонстрируя полный цикл от заказа через API до делегирования энергии и перевода USDT с нулевым сжиганием TRX за энергию.

В этих транзакциях основной сети переводы USDT, которые стоили бы 27,30 TRX через механизм сжигания, обошлись в 1,43 TRX через маршрутизированную аренду энергии MERX. Транзакции делегирования и перевода публично проверяемы в любом обозревателе блоков TRON.

Это не симуляции в тестовой сети. Это реальные транзакции основной сети с реальными TRX и реальными USDT, демонстрирующие, что маршрутизация, интеграция с поставщиками и исполнение в блокчейне работают в продакшене.

## Для кого создан MERX

**Платёжные процессоры**, отправляющие USDT мерчантам, фрилансерам или поставщикам. Стоимость энергии каждого перевода - это прямой удар по маржинальности.

**Криптовалютные биржи**, обрабатывающие выводы TRC-20. Затраты на энергию либо поглощаются (снижая прибыль), либо перекладываются на пользователей (снижая конкурентоспособность).

**Торговые боты**, выполняющие частые переводы в блокчейне в рамках стратегий арбитража или маркетмейкинга. Каждая доля TRX, сэкономленная за перевод, масштабируется с объёмом.

**DeFi-протоколы**, взаимодействующие со смарт-контрактами TRON. Затраты на энергию влияют на экономику каждой операции в блокчейне.

**Команды управления казначейством**, которым необходимо оптимизировать операционные расходы на перемещение USDT и других токенов TRC-20 в блокчейне.

**AI-агенты**, автономно управляющие операциями в блокчейне и нуждающиеся в программном способе приобретения энергии по лучшей цене.

## Начало работы

Посетите [merx.exchange](https://merx.exchange), чтобы создать аккаунт и увидеть рынок энергии в реальном времени. Платформа работает и обрабатывает заказы в основной сети TRON.

Для интеграции через API начните с [документации](https://merx.exchange/docs). Установите SDK для вашего языка:

```bash
# JavaScript / TypeScript
npm install merx-sdk

# Python
pip install merx-sdk
```

Для интеграции AI-агентов:

```bash
npm install merx-mcp
```

Первый шаг всегда одинаков: проверьте текущие цены. Посмотрите, как выглядит рынок. Сравните, сколько вы сейчас платите за энергию, с тем, что предлагает агрегированный рынок. Цифры говорят сами за себя.
